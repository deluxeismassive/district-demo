**Prerequisite 1: GitHub Pages enabled on the repo**
- URL: `https://github.com/deluxeismassive/district-demo/settings/pages`
- Required setting: Source = "Deploy from a branch", Branch = `gh-pages`, Folder = `/ (root)`
- If not enabled before Task 1: the first `gh-pages` push will create the branch, but the live URL will return 404 until Pages is enabled in repo settings.

**Prerequisite 2: Git authentication available locally**
- The `gh-pages` CLI uses `git push` under the hood
- The developer running `npm run deploy` MUST have either:
  - HTTPS credentials cached (e.g. GitHub CLI `gh auth status` reports authenticated; or git credential helper has stored a PAT), OR
  - SSH key configured for `git@github.com`
- Verify with: `git ls-remote origin` (should NOT prompt for credentials)

**Expected timeline:**
- `nuxi generate`: ~30-60s
- `gh-pages -d .output/public --nojekyll`: ~5-30s push
- GitHub Pages CDN propagation: 1-3 min typical, up to 20 min worst case
- `node scripts/smoke.mjs`: up to 100s (5 probes × up to 5 retries × 20s) if first attempt fails
- **Total: 2-6 minutes typical; up to ~10 minutes if CDN is slow**

**What success looks like:**
- `npm run deploy` exits 0
- Smoke output ends with: `5/5 probes passed`
- Live URL `https://deluxeismassive.github.io/district-demo/` returns HTTP 200 with the Dashboard pre-rendered HTML

    - Nuxi build summary (routes pre-rendered, output size)
    - `Published to origin/gh-pages` (or similar from gh-pages CLI)

    - Process exits 0

    **If the script exits 1:**
    - Note which probe failed and on what reason (`HTTP 404` vs `marker not found` vs `fetch failed`)
    - Wait 2 minutes and run `node scripts/smoke.mjs` directly (without re-deploying)
    - If still failing: check `https://github.com/deluxeismassive/district-demo/settings/pages` and confirm:
      - Source: "Deploy from a branch"
      - Branch: `gh-pages`, Folder: `/ (root)`
    - If `HTTP 404` on `/`: the `gh-pages` branch may not have been created. Check `git ls-remote origin gh-pages` — if empty, the push failed silently. Re-run `npm run deploy`.
    - If `marker not found`: the live HTML doesn't contain the static h1/h2 text. Pull the live HTML with `curl -s https://deluxeismassive.github.io/district-demo/ | head -200` and inspect.

    - Total elapsed time (record from `time npm run deploy` if shell supports it; otherwise estimate)
    - Number of WAIT cycles per probe (proxy for CDN propagation speed)
    - Final smoke output (5/5 line)
    - The first 10 lines of `curl -s https://deluxeismassive.github.io/district-demo/` (verifies SSR HTML actually serves)
