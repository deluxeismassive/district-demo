# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Shell & Routing.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (PrimeVue 4 + Tailwind CSS v4) |
| Preset | PrimeVue Aura theme, customized via `definePreset()` with `#484CE6` as primary |
| Component library | PrimeVue 4.5.5 (`primevue` + `@primeuix/themes`) |
| Icon library | PrimeIcons 7.0.0 (`primeicons`) |
| Font | System UI stack: `system-ui, 'Segoe UI', Roboto, sans-serif` |

**Source:** D-09 (PrimeVue + Tailwind), D-10/D-11 (brand colors), RESEARCH.md Standard Stack. Aura theme chosen per Claude's Discretion — pairs cleanest with `#484CE6` at cool blue-violet hue. PrimeIcons chosen per Claude's Discretion — consolidates icon system with PrimeVue internals (DataTable sort icons use PrimeIcons font regardless).

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap in nav items |
| sm | 8px | Nav item list spacing (`space-y-1`), compact inline gaps |
| md | 16px | Default content padding unit, nav item padding (`px-3 py-2`) |
| lg | 24px | Page content padding (`p-6` = 24px), sidebar brand header padding (`px-6 py-5`) |
| xl | 32px | Dashboard skeleton card grid gap |
| 2xl | 48px | Not used in Phase 1 |
| 3xl | 64px | Not used in Phase 1 |

Exceptions:
- Sidebar brand header vertical padding: 20px (`py-5`) — closest even multiple that gives logo area visual weight
- Nav item border-bottom (`border-b`): 1px — structural, not spacing scale
- Touch target minimum: not applicable (desktop-only demo per REQUIREMENTS.md out-of-scope)

## Typography

| Role | Size | Weight | Line Height | Element |
|------|------|--------|-------------|---------|
| Body | 14px | 400 (regular) | 1.5 | Skeleton view labels, nav item labels |
| Label | 12px | 500 (medium) | 1.4 | District name in header, section meta text |
| Heading | 20px | 600 (semibold) | 1.2 | Page title (`h1`) in each stub view |
| Brand | 18px | 600 (semibold) | 1.0 | "Schoolday" in sidebar header |

Font stack (all roles): `system-ui, 'Segoe UI', Roboto, sans-serif`

**Source:** RESEARCH.md Pattern 4 (nav uses `text-sm font-medium` = 14px/500); Pattern 7 stub view headings use `text-xl font-semibold` = 20px/600. Brand name from Pattern 4 (`text-lg font-semibold` = 18px/600). Label role is Claude's Discretion — 12px/500 is standard for supporting metadata in enterprise SaaS.

No serif or display type in Phase 1. The skeleton phase has no prose content requiring a distinct display size.

## Layout Contract

### AppShell

- Full viewport height: `h-screen`
- Flex row, no wrap: `flex overflow-hidden`
- Sidebar and content never overlap — sidebar is `shrink-0`

### Sidebar

| Property | Value | Source |
|----------|-------|--------|
| Width | 256px (`w-64`) | Claude's Discretion — RESEARCH.md open question resolved: `w-64` fits all 4 nav labels with icon |
| Height | 100vh (`h-screen`) | Fixed full height |
| Background | `#111827` (`bg-gray-900`) | D-08, RESEARCH.md Pattern 4 |
| Position | Static in flex row (no `position: fixed`) | Achieved via flex shrink-0 — content area scrolls independently |
| Overflow | Hidden on sidebar itself; sidebar content does not scroll | All 4 nav items + brand header fit within any reasonable viewport |

### Sidebar Brand Header

| Property | Value |
|----------|-------|
| Content | Text "Schoolday" (no icon/logo in Phase 1) |
| Padding | `px-6 py-5` (24px horizontal, 20px vertical) |
| Border | `border-b border-gray-700` (1px separator below brand) |
| Text | 18px, semibold, white (`text-lg font-semibold text-white`) |

**Source:** D-06 ("Schoolday" in sidebar header), RESEARCH.md Pattern 4. No logo SVG asset exists yet — text-only brand mark in Phase 1.

### Nav Items

| State | Background | Text Color | Behavior |
|-------|-----------|-----------|---------|
| Default | Transparent | `#9CA3AF` (gray-400) | — |
| Hover | `bg-gray-800` | `text-white` | CSS `hover:` transition-colors |
| Active | `bg-primary` (`#484CE6`) | `text-white` | `RouterLink active-class` auto-applied |

Nav item anatomy (each `<li>` → `<RouterLink>`):
- Padding: `px-3 py-2` (12px horizontal, 8px vertical)
- Border radius: `rounded-lg` (8px)
- Icon: PrimeIcons `<i>` element, `text-base` size (16px), positioned left
- Label: `text-sm font-medium` (14px, 500), flex-1
- Icon-label gap: `gap-3` (12px)

Nav item icons per route:
| Route | Label | Icon class |
|-------|-------|-----------|
| `/` | Dashboard | `pi pi-home` |
| `/discovery` | Discovery | `pi pi-search` |
| `/reports` | Reports | `pi pi-chart-bar` |
| `/settings` | Settings | `pi pi-cog` |

Active class note: Dashboard route (`/`) uses `exact-active-class` to prevent it staying highlighted on all child paths. All four routes receive both `active-class` and `exact-active-class` set to `bg-primary text-white` — safe on all four.

Nav list padding: `py-4 px-3` on the `<ul>`, `space-y-1` between items.

**Source:** RESEARCH.md Pattern 4 (verbatim structure), D-05 (nav order), D-04 (fixed sidebar).

## Loading Skeleton Contract

All stub views use PrimeVue `<Skeleton>` component. Import per-component: `import Skeleton from 'primevue/skeleton'`.

PrimeVue Skeleton renders a shimmer-animated gray block. No custom animation needed.

### DashboardView — Stat Card Skeletons

Layout: 4-column grid (`grid grid-cols-4 gap-4`), each cell a card-like block.

| Element | Width | Height | Notes |
|---------|-------|--------|-------|
| Stat card block | `100%` | `96px` | `border-radius="8px"` — card shape |

Row count: 1 row of 4 cards. Represents the summary stat cards the Dashboard will show in Phase 2+.

Below the card row: 2 skeleton rows of narrower blocks (`width="60%" height="1rem"`) representing a secondary summary section.

### DiscoveryView — Table Skeleton

Layout: Vertical stack with simulated table header + 8 data rows.

Table header row (1x):
| Column | Width | Height |
|--------|-------|--------|
| Vendor name col | `200px` | `1rem` |
| Category col | `120px` | `1rem` |
| Usage col | `100px` | `1rem` |
| Status col | `100px` | `1rem` |

Data rows (8x, `v-for="i in 8"`), `flex gap-4 py-3 border-b border-gray-100`:
| Column | Width | Height | Border radius |
|--------|-------|--------|--------------|
| Vendor name | `200px` | `1rem` | default |
| Category | `120px` | `1rem` | default |
| Usage | `100px` | `1rem` | default |
| Badge (tag/status) | `80px` | `1.5rem` | `9999px` (pill) |

### ReportsView — Tab Bar + Table Skeleton

Tab bar is NOT a skeleton — it is a real interactive element in Phase 1 (tabs must work to demonstrate the in-page nav pattern).

Tab bar structure: `flex gap-1 border-b border-gray-200 mb-6`

| Tab | Default state | Active state |
|-----|--------------|-------------|
| DPA | `border-transparent text-gray-500 hover:text-gray-700` | `border-primary text-primary border-b-2` |
| Risk Position | same | same |
| 1EdTech | same | same |

Default active tab: `DPA` (first tab, `ref('DPA')`).

Below tab bar: 6 skeleton data rows, same column structure as DiscoveryView (vendor + date + badge).

### SettingsView — List Row Skeletons

Layout: Vertical list of label+value pairs.

| Element | Structure | Width | Height |
|---------|-----------|-------|--------|
| List row | `flex justify-between py-3 border-b border-gray-100` | full | auto |
| Label side | `Skeleton` | `140px` | `1rem` |
| Value side | `Skeleton` | `200px` | `1rem` |

Row count: 6 rows. Represents settings fields that will appear in Phase 6 (Tags Management page settings).

Page title in each stub view: `<h1 class="text-xl font-semibold mb-6 text-gray-900">{Section Name}</h1>` before skeleton content.

## Component Map

| UI Element | PrimeVue Component | Notes |
|------------|-------------------|-------|
| Loading skeleton blocks | `<Skeleton>` from `primevue/skeleton` | Per-component import, no global registration |
| Tab buttons in ReportsView | Native `<button>` with Tailwind classes | PrimeVue `<TabMenu>` is an alternative but requires route integration; plain buttons are simpler for in-page tab state |

## CSS Token Declarations

All tokens defined in `src/style.css` via Tailwind v4 `@theme` block:

```css
@import "tailwindcss";

@theme {
  --color-primary: #484ce6;
  --color-accent: #da8231;
  --color-sidebar: #111827;
}
```

Generated utility classes available after this declaration:
- `bg-primary`, `text-primary`, `border-primary` → `#484CE6`
- `bg-accent`, `text-accent` → `#DA8231`
- `bg-sidebar` → `#111827`

Existing `src/style.css` CSS custom properties (Vue starter defaults: `--text`, `--bg`, `--accent`, etc.) are replaced entirely. The new file begins with `@import "tailwindcss"` and the `@theme` block. No legacy starter styles carry forward.

**Source:** RESEARCH.md Pattern 5.
