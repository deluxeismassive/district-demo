## Data Contract

Source: CONTEXT.md D-01 through D-15.

### Risk Tier Formula (computed in component)

```js
function calcTier(dpa, discovery, edtech) {
  // Base tier from DPA status + usage volume
  const badDpa = dpa?.status === 'Unsigned' || dpa?.status === 'Expired'
  const highUsage = (discovery?.userCount ?? 0) > 1000
  let tier = badDpa
    ? (highUsage ? 'High' : 'Medium')
    : 'Low'

  // 1EdTech modifier: active Certified reduces tier by one level
  if (edtech?.certificationStatus === 'Certified') {
    if (tier === 'High') tier = 'Medium'
    else if (tier === 'Medium') tier = 'Low'
    // Low stays Low
  }

  return tier
}
```

## Claude's Discretion Areas

Source: CONTEXT.md "Claude's Discretion" section.

These four areas are not locked by user decisions and may be resolved by the executor:

1. **Donut option object structure** — Use the ECharts option object in the Data Contract section above. Legend at bottom, no segment labels, tooltip with count and percent.

2. **Vendor name as drawer link** — The VendorDrawer pattern exists in DiscoveryView. For demo value, clicking a vendor row in the risk table SHOULD open the VendorDrawer — the sales rep can then drill into any flagged vendor immediately. Implement `@row-click="onRowClick"` on the DataTable and pass the enriched row (with vendorId) to VendorDrawer. This reuses the existing VendorDrawer component with zero new code.

3. **Chart card styling** — Use `bg-white rounded-lg border border-gray-200 p-6 mb-6` container. Chart height 280px. This matches the card-like containers used elsewhere in the portal.

4. **"Users" column header** — Use "Users" (singular label matching the DiscoveryView column header for consistency).
