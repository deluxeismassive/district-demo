## Spacing Scale

Declared values (multiples of 4 only). Implemented via Tailwind v4 utility classes.

| Token | Value | Tailwind class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, tag pill inner padding |
| sm | 8px | `p-2` / `gap-2` | Compact element spacing, button icon gap |
| md | 16px | `p-4` / `gap-4` | Default element spacing, table cell padding |
| lg | 24px | `p-6` / `gap-6` | View-level padding (matches existing `p-6` in stub views) |
| xl | 32px | `p-8` / `gap-8` | Section breaks within slide-over panel |
| 2xl | 48px | `py-12` | Major section breaks |
| 3xl | 64px | `py-16` | Not used in this phase |

Exceptions:
- Filter input row: `mb-4` (16px) between input and table — standard md
- Drawer panel: PrimeVue Drawer default padding retained; do not override with custom padding on the root element
- Tag pills: PrimeVue Tag default padding retained — do not override with custom padding on the Tag root element

> Source: Existing `p-6` in DiscoveryView stub and AppShell header `pl-6`; 8-point scale default

## Typography

|------|------|----------------|--------|----------------|-------------|

## Interaction Contract

### Discovery table

| Interaction | Behavior |
|-------------|----------|
| Click column header | Sort ascending → sort descending → clear sort |
| Type in filter input | Real-time global filter across Vendor Name and Category columns |
| Click a vendor row | Open VendorDrawer (slide-over from right); row stays visually highlighted |
| Hover row | `bg-gray-50` row highlight (PrimeVue DataTable default) |

### Vendor detail slide-over (VendorDrawer)

| Interaction | Behavior |
|-------------|----------|
| Open | Slides in from right; backdrop dims main content (PrimeVue Drawer default modal behavior) |
| Select tags in MultiSelect | Immediately updates `useTagsStore.assignments[vendorId]`; tags column in table reflects change |
| Remove tag chip in MultiSelect | Same store update as above |
| Close (X button or backdrop click) | Drawer closes; table remains sorted/filtered in same state |

### Tag pills in table

| Interaction | Behavior |
|-------------|----------|
| Display | Child tag pills rendered inside Tags column; pills grouped visually by parent group |
| Tag pill color | Matches parent group color from `useTagsStore.tagGroups[n].color`; fallback to `#da8231` if no color |
| No click action | Tag pills in the table are read-only; clicking opens nothing |

## Data File Contract

Phase 2 creates four files in `src/data/`. Each file is a plain ES module `export default []` or `export default {}`.

| File | Shape | Notes |
|------|-------|-------|
| `src/data/vendors.js` | `Array<Vendor>` | `{ vendorId, name, category, privacyScore: { informationCollected, useOfInformation, dataSharing, securityMeasures, userRights, retentionPeriod, complianceWithLaws, updatesToPolicy, clarityAndTransparency, contactInformation } }` |
| `src/data/discovery.js` | `Array<DiscoveryRecord>` | `{ vendorId, frequency, lastSeen, userCount, studentCount }` — keyed by vendorId for join |
| `src/data/dpa.js` | `Array<DpaRecord>` | `{ vendorId, status, signedDate, expiryDate }` — placeholder; Phase 3 builds view |
| `src/data/edtech.js` | `Array<EdtechRecord>` | `{ vendorId, certificationStatus }` — placeholder; Phase 4 builds view |

Fixture count: 25–30 realistic edtech vendor names.
Join key: stable `vendorId` string (`"vendor-google-classroom"` format) — never display name.
