```typescript
function setVendorTags(vendorId: string, tagIds: string[]) {
  if (tagIds.length === 0) {
    delete assignments.value[vendorId]
  } else {
    assignments.value[vendorId] = tagIds
  }
}

function clearVendorTags(vendorId: string) {
  delete assignments.value[vendorId]
}

return { tagGroups, assignments, setVendorTags, clearVendorTags }
```

Nuxt UI v4 USelectMenu API (research §6 — verify-source-of-truth: the `items` shape is array-of-arrays for groups, NOT [{label, items}]):
```typescript
// items prop: Array | Array<Array> — array-of-arrays creates separated groups.
// value-key: bind to a specific property (e.g., 'id') instead of the whole object.
// label-key: display field (e.g., 'name'); default 'label'.
// multiple: enables multi-select; v-model is string[] (when value-key='id').
// Anti-pattern (PrimeVue port, research Pitfall #5): [{ label: 'Group', items: [...] }] — DOES NOT WORK in v4.
```

WHY the explicit `:model-value` + `@update:model-value` pattern instead of `v-model` (research §6):
- The `?? []` fallback in `:model-value` is safer than letting v-model construct an undefined-indexed ref (assignments[vendorId] is undefined when no tags assigned).
- Surfaces a clear named action `setVendorTags` for grep verification.
- Lets the store action also delete empty assignments (keeps localStorage JSON tidy — research §7).
