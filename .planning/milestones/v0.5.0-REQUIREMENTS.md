# Requirements Archive: v0.5.0 MVP

**Archived:** 2026-05-21

## v1 Requirements — All Complete

### Foundation

- [x] **FOUND-01**: App has multi-page SPA routing with a persistent sidebar nav linking all sections
  - *Outcome: Validated — Vue Router hash history, SidebarNav driven by route meta*
- [x] **FOUND-02**: Every page renders inside a consistent app shell (header, nav, content area)
  - *Outcome: Validated — AppShell.vue wraps RouterView on every route*
- [x] **FOUND-03**: All display data lives in `src/data/*.js` files, editable without touching components
  - *Outcome: Validated — 4 data files: vendors.js, discovery.js, dpa.js, edtech.js*
- [x] **FOUND-04**: Pinia store manages shared tag state accessible across all pages
  - *Outcome: Validated — useTagsStore() with localStorage persistence*

### Discovery

- [x] **DISC-01**: User can view a sortable, filterable table of vendors with name, category, usage metrics, and user/student counts
  - *Outcome: Validated — 27-vendor PrimeVue DataTable with 7 sortable columns*
- [x] **DISC-02**: User can assign and remove tags on a vendor from the Discovery table
  - *Outcome: Validated — grouped MultiSelect in VendorDrawer, assignments persisted*
- [x] **DISC-03**: User can expand or open a detail view for a vendor to see full usage data
  - *Outcome: Validated — VendorDrawer slide-over with 10-axis ECharts radar chart*

### DPA

- [x] **DPA-01**: User can view a sortable, filterable table of vendors with DPA status badge, signed date, and expiry date
  - *Outcome: Validated — DpaGrid 6-column table with color-coded status badges*
- [x] **DPA-02**: Page header shows a count of vendors that need DPA review
  - *Outcome: Validated — Dashboard "Top 8 Vendors Needing Attention" card + KPI tiles*

### Risk

- [x] **RISK-01**: Each vendor has a calculated risk tier (High / Medium / Low) derived from DPA status and usage volume
  - *Outcome: Validated — calcTier formula (DPA status + usage + 1EdTech modifier)*
- [x] **RISK-02**: A donut chart displays the distribution of vendors by risk tier
  - *Outcome: Validated — ECharts donut chart (High:2 / Medium:7 / Low:18)*

### 1EdTech

- [x] **EDTECH-01**: User can view a table of vendors with their 1EdTech certification status badge
  - *Outcome: Validated (adjusted scope) — status badge in VendorDrawer, not standalone table*

### Tags

- [x] **TAGS-01**: User can create, rename, and delete tags with color assignment on a Tags management page
  - *Outcome: Validated — full CRUD SettingsView with inline rename, 8-color palette, cascade delete*
- [x] **TAGS-02**: User can assign and remove tags on vendor rows in the Discovery page
  - *Outcome: Validated — grouped MultiSelect in VendorDrawer, localStorage persistence*
