> **Note on naming:** This milestone was originally tracked as `v1.0.0 Nuxt Migration` throughout planning and execution. It was renamed to `v0.6.0` at `/gsd:complete-milestone` time because 1.0 is reserved for "production-ready with real backend / auth," and this milestone ‚Äî while a complete framework rewrite ‚Äî still ships with fully synthetic data. ## Overview

Full framework migration from Vue 3 + Vite SPA + PrimeVue 4 ‚Üí Nuxt 4 SSR + Nuxt UI v4 + TypeScript throughout. All v0.5.0 demo features preserved with zero behavior regressions. Mock data moved from client-side `src/data/*.js` to server-side typed `server/data/*.ts` served through Nitro `server/api/` routes. Static site generation via `nuxi generate` published to GitHub Pages; `nuxt.config.ts` structured with a 3-line glidepath for future AWS Amplify SSR migration.

## Milestone Summary

**Key Decisions:**

| Decision | Rationale |
|----------|-----------|
| Nuxt 4 SSR over SPA mode | Supports secure server-side API calls (future app-catalog backend); `nuxi generate` produces static output for GitHub Pages now |
| Nuxt UI v4 (NOT v3) | Latest stable when scaffold landed; Tailwind v4 native; superior TS types via installed `.d.ts` |
| Mock data ‚Üí `server/data/*.ts` (typed) served via `server/api/` Nitro routes | Establishes the API contract that real backend calls will follow; eliminates v0.5.0 client-side join overhead |
| ECharts SSR strategy: `<ClientOnly>` for interaction-mounted charts; initial-SSR for static charts (donut) | Avoids "window is not defined" SSR crash on drawer radar; donut renders via nuxt-echarts placeholder + hydrates |
| UBadge color strategy: `:style` hex injection from `shared/utils/riskLabels.ts` (NO Tailwind classes) | Sales-rep can recolor any badge by editing one hex value in one file; survives SSR (verified hex form, not `rgb()`) |
| `useFetch` URL-key dedup as the contract (no manual `key` option) | One cache entry shared across multiple components fetching the same URL ‚Äî verified empirically across 3 call sites of `/api/dpa` |
| Pinia setup-store action pattern (named functions inside callback + included in return) | Grep-able write paths; `setVendorTags` empty-array delete branch enables cascade-delete reuse |
| `pinia-plugin-persistedstate` with `storage: 'localStorage'` for Pinia SSR | Tag state survives hard refresh via localStorage roundtrip ‚Äî verified during Phase 12 UAT |
| `--nojekyll` flag on every `gh-pages` deploy | Mandatory ‚Äî without it, dotfile is dropped, Jekyll hides `_nuxt/` chunks, live site renders blank |
| Inline 3-line Amplify glidepath comment in `nuxt.config.ts` + expanded ADR at `.planning/adr/AMPLIFY-GLIDEPATH.md` | Code is canonical for the switch recipe; ADR is canonical for the migration reasoning ‚Äî both checked at DEPLOY-02 readability gate |
| Manual `npm run deploy` (no GitHub Actions) | Solo sales-demo workflow favors human-controlled timing over CI ceremony ‚Äî matches v0.5.0 pattern |
